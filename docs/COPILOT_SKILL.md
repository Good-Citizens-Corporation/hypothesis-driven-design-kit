# Copilot Skill: Hypothesis-Driven Design

This repo includes a `.skill` file that teaches GitHub Copilot the HDD methodology.
When installed, Copilot can analyze artifacts, generate hypotheses, create test plans,
and produce GitHub-ready outputs — all following the HDD discipline.

---

## Installation

### Option 1: Skill Import (Recommended)

1. Open VS Code with Copilot Chat enabled
2. Open the Copilot Chat panel
3. Type: `@workspace /installSkill`
4. Select `hypothesis-driven-design.skill` from this repo
5. The skill activates immediately

### Option 2: Manual Install

1. Unzip the skill file:
   ```bash
   unzip hypothesis-driven-design.skill -d .github/copilot-skills/
   ```
2. Copilot detects the skill automatically on next chat

## When the Skill Activates

The skill triggers automatically when you share:

| Input type | Examples |
|------------|----------|
| Strategy artifacts | PRDs, briefs, strategy docs, roadmaps |
| Research artifacts | Journey maps, persona sets, research reports |
| Presentation decks | PPTX/PDF strategy decks, pitch decks |
| Hypothesis work | Writing, critiquing, or formatting hypotheses |

**Rule of thumb:** If the artifact makes claims about users, value, or outcomes — the skill applies.

## What It Does

The skill runs a 5-phase analysis on any artifact:

```
1. NARRATIVE CHALLENGE    → Surface unsupported assertions
2. ASSUMPTION EXTRACTION  → Extract hidden assumptions
3. CONFIDENCE RANKING     → Rank by consequence of being wrong
4. HYPOTHESIS GENERATION  → Convert to canonical hypothesis format
5. ARTIFACT ROUTING       → Generate the output you need
```

### Canonical Hypothesis Format

Every output traces back to this atomic unit:

> *We believe [outcome] is achievable if people doing/being [situation] can get [motivation].*

If you can't write this sentence for what you're building, you're not doing product work.

## The Fidelity Ladder (Test Plans)

When you request a test plan, the skill generates a **progressive validation roadmap**:

| Fidelity | Method | Cost | Signal |
|----------|--------|------|--------|
| 0 | Observe existing behavior | Zero | Baseline |
| 1 | Conversation / survey | Hours | Stated preference |
| 2 | Concierge (manual delivery) | Days | Real behavior |
| 3 | Wizard of Oz (fake backend) | Week | Behavior at scale |
| 4 | Prototype / thin slice | Sprint | Interaction data |
| 5 | MVP | Sprints | Outcome data |
| 6 | Full build | Months | Scale validation |

**The rule:** Never escalate to fidelity N+1 until fidelity N produces signal above your confidence threshold.

## Key Concepts

### Narrative vs. Evidence
A **narrative** is a claim presented as a finding with no source, no measurement, and no way to prove it wrong.
Narratives feel like evidence but aren't. The skill identifies and flags them.

### Stopping Conditions
Every hypothesis needs a **stopping condition** — what would cause you to abandon it.
Without one, you're not testing. You're fishing.

### INVEST Compliance
Slices generated by the skill follow INVEST:
- **I**ndependent — can be validated without other hypotheses
- **N**egotiable — implementation is flexible; outcome being tested is not
- **V**aluable — advances validation of parent hypothesis
- **E**stimable — assumption is specific enough to scope
- **S**mall — tests one assumption, not a cluster
- **T**estable — has observable acceptance criteria

## Troubleshooting

### Skill not triggering?
- Ensure the skill is installed (check Copilot Chat settings)
- Try an explicit prompt: "Using the hypothesis-driven-design skill, analyze this PRD"

### Output missing stopping conditions?
The skill enforces output contracts. If your input doesn't contain enough information to generate
a stopping condition, it will ask for clarification rather than guess.

### Input is too thin?
If you provide a single sentence or brief claim instead of a full artifact, the skill will:
1. Flag that Phase 1 cannot decompose it (the input IS the narrative)
2. Either ask for the source document, or synthesize the likely artifact and proceed

This is intentional — a single assertion has nothing to analyze. Either provide the full document
or let the skill reconstruct what the document probably says.

### PPTX files?
Extract text first:
```bash
python -m markitdown presentation.pptx
```
Then paste the output into chat.
